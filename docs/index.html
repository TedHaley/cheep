What is it?

cheep is a tiny, single-binary, interactive multi-agent coding shell for your terminal. A lead orchestrator agent breaks a task into pieces, hands each to one or more executor agents (in parallel, each in its own isolated git worktree), validates the work before it merges, and recovers any executor that gets stuck вҖ” loops, runs out of context, or errors out. You watch it happen, with a tab per agent.

The whole point: the best results for the least money. Spend premium tokens only where judgment matters, run the rest on cheap or local models, and watch the token-and-dollar meter so you always know what a task actually cost.

Orchestrator
Plans, delegates, and verifies. Reads files and runs tests to check the work вҖ” but never edits code itself. Best with a strong model.
Executors
Do the actual editing and running, in parallel. Point them at free local models and the work costs nothing.

Provider-agnostic. Any role can be Anthropic (Claude) or any OpenAI-compatible endpoint вҖ” local (Ollama, LM Studio, vLLM, llama.cpp) or cloud (OpenAI, Grok/xAI, DeepSeek, Groq, OpenRouter, Mistral, вҖҰ). Mix an expensive planner with free local workers, or run the whole thing locally.

Why cheep?

  • Watch every agent. A tab per agent with live output; see what's delegated to whom and step in any time.
  • See every dollar. A live counter shows tokens and estimated cost per agent (per-token prices from the maintained LiteLLM dataset), marks local work free, and tallies what you saved versus running it all on a premium model.
  • Cheap-first escalation. Each subtask runs on a cheap or local model first; only if it fails, loops, or runs out of context does cheep retry on a pricier one вҖ” so you pay premium rates only for the work that truly needs them.
  • Validated before it merges. Every delegated subtask runs in a private git worktree; your project's declared checks run and a fresh-context reviewer judges the diff (with bounded automatic fix rounds) before anything touches your checkout. Failed work stays on its branch вҖ” never lost.
  • NEW No-mistakes mode. /nomistakes on: every shared write and command asks first, and nothing merges until you approve the branch diff. Headless runs hold everything on branches вҖ” cheep becomes physically incapable of surprising you.
  • NEW Scouts and ships. The orchestrator marks each subtask ship (delivers changes through the merge machinery) or scout (research/audit вҖ” changes discarded, findings saved as a report). Research on free local models gets even cheaper.
  • NEW Ship straight to PRs. /delivery pr pushes each validated branch and opens a pull request instead of merging locally вҖ” safe on shared repos; your checkout is never modified.
  • NEW Session tree. /fork branches the conversation from any earlier turn (shared context kept, nothing re-paid); /tree navigates every session with forks nested under parents.
  • NEW Reasoning per role. Append :low|:medium|:high to any model name вҖ” extended thinking on Claude, reasoning_effort elsewhere. Planner thinks hard, workers stay cheap, in one suffix.
  • NEW Pi extensions. Run pi coding agent extensions from npm inside cheep вҖ” cheep pi add <pkg> and their tools join your agents' tool set.
  • NEW Crash-proof delegation. Every delegation writes a marker while it runs; if cheep dies mid-flight, the next launch tells you exactly what was interrupted and which branch holds the partial work.
  • Per-project budgets. Set a dollar ceiling per project вҖ” cheep warns at 80% and stops at 100% вҖ” and the orchestrator routes to the cheapest capable executor to stay within it. Cost is bounded, not just visible.
  • Loop until green. An iterate_until primitive re-runs a subtask on a cheap executor and re-runs your check (tests, build, lint) until it passes вҖ” bounded by rounds and budget. Loops on cheap models are nearly free, so iterate aggressively.
  • Project launchpad. cheep init sets up any repo for agentic work: AGENTS.md with detected build/test/lint checks, lessons that persist corrections, open-standard skills вҖ” readable by Claude Code, Codex, and friends too.
  • Prompt templates & skills. Markdown templates invoked as /name args; knowledge files agents load on demand instead of bloating every prompt.
  • Modes & approvals. shift+tab cycles chat вҶ’ plan вҶ’ auto; /approval gates risky calls with diff previews; /stow sweeps lessons and a handoff note to disk before a reset.
  • Tools & MCP. Built-in file/shell tools, plus MCP servers (stdio or HTTP) for every agent.
  • One static binary. No runtime, no dependencies. brew install, go install, or download. Every feature above is covered by the test suite (go test ./...).

1 В· A parallel scout swarm audits your repo for pennies

Scouts investigate in parallel on free local models. Their file changes are discarded automatically; each one's findings come back to the orchestrator and land as a report in ~/.cheep/reports/.

# the orchestrator fans out three kind:"scout" subtasks in ONE delegate call,
# one tab per scout; reports saved under ~/.cheep/reports/, checkout untouched

2 В· Fix-until-green, nearly free

Anything with an objective pass/fail signal can be looped on a cheap model. Each failure is fed back in; the loop is bounded by rounds and your budget.

вҖә the tests in ./internal/parser are failing вҖ” fix them until `go test ./internal/parser`
  passes

# orchestrator calls iterate_until{subtask, check:"go test ./internal/parser"}
# on the cheapest executor; rounds are ~free on a local model, budget caps the worst case

Tried something and want to compare it against another approach вҖ” without re-paying for all the context the session already built up? Fork it.

8 В· Walk away mid-task. Come back on your phone.

Run cheep inside tmux, and the same live session вҖ” running task, tabs, cost meter вҖ” is reachable from your phone over Tailscale SSH. If anything ever dies mid-delegation, the next launch says exactly what was interrupted and where the partial work lives.

tmux new -As cheep && cheep    # on your Mac
вҖә refactor the storage layer; run the full test suite when done
# вҖҰ later, from your phone:
ssh my-macbook -t 'tmux attach -t cheep'

Full setup (Tailscale, mosh, phone keyboards): the portable guide. Before you step away, /stow writes lessons and a done / in-flight / next-steps note to disk.

Best of all worlds

You don't have to pick one model and live with its trade-offs. cheep lets you put the right model on each job: the orchestrator only needs to be smart enough to plan and check the work, and each executor only needs to be good enough for the task in front of it вҖ” so you spend premium tokens on judgment and run the bulk of the work cheap, or free and local.

Smart planner, cheap hands
Let a frontier model (Claude, GPT) plan, delegate, and verify, while local models do the editing, searching, and test runs вҖ” where they're more than good enough вҖ” at no cost.
Right model for the job
Send code to a coding-tuned model, research or drafts to a fast general one, big files to a long-context one. The orchestrator sees each executor's cost tier and your project budget, and routes cheapest-capable-first вҖ” escalating only when needed. Steer it with natural-language rules in .cheep/dispatch.json.
All local, zero cost
Point every role at local models for private, offline work that costs nothing вҖ” you still get planning, parallelism, validation, and verification on top.
Spend where it counts
A live token counter splits premium vs free usage, so you keep expensive tokens for the hard reasoning and let the long tail of grunt work run cheap.

Example: one planner, two specialists

A Claude orchestrator plans and verifies (thinking hard); a free local Qwen does the heavy lifting; a cloud DeepSeek handles anything that needs a bit more horsepower. This is just ~/.cheep/config.json вҖ” but you'd normally build it in the configurator:

{
  "orchestrator": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-6:high"
  },
  "executors": [
    {
      "name": "local-qwen",
      "provider": "openai",
      "endpoint": "http://127.0.0.1:1234/v1",
      "model": "qwen/qwen3.6-35b-a3b"
    },
    {
      "name": "deepseek",
      "provider": "openai",
      "endpoint": "https://api.deepseek.com/v1",
      "model": "deepseek-chat",
      "api_key": "sk-..."
    }
  ]
}

The Claude orchestrator needs no api_key here вҖ” it's picked up from ANTHROPIC_API_KEY in ~/.cheep/keys.env or your shell. The orchestrator decides which executor to hand each subtask to.

Quick start

# 1. (optional) run a local model вҖ” any OpenAI-compatible server works
#    e.g. Ollama, LM Studio, vLLM, llama.cpp
ollama serve &
ollama pull qwen2.5-coder

# 2. launch cheep
cheep

On first launch cheep opens a setup configurator: it scans your machine for running model servers and API keys, and you pick an orchestrator and at least one executor вҖ” or enter cloud credentials manually (Anthropic, OpenAI, Grok, DeepSeek, вҖҰ). Then you're in the shell. Reopen it any time with /config.

Setting up a project for agentic work? cheep init scaffolds AGENTS.md with detected validation checks. Prefer one-shot, non-interactive runs?

cheep run "add a /health endpoint and a test for it" --workdir .   # add --json for CI

Commands

There are no required environment variables. cheep stores its setup in ~/.cheep/config.json and API keys in ~/.cheep/keys.env (0600). You configure agents interactively, not by hand.

CommandWhat it does
cheepInteractive shell (opens setup on first run)
cheep run "<task>" [--json]Run a single task non-interactively (JSONL events for CI)
cheep initSet up a project: AGENTS.md, detected checks, toolbelt skill
cheep validate [--review]Run the project's checks (+ optional AI review of the diff)
cheep worktree вҖҰPooled, artifact-warm git worktrees (usable from other harnesses)
cheep pi add <pkg>Install a pi.dev extension; its tools join every agent
cheep checkPing the orchestrator and every executor
