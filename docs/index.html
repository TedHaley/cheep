What is it?

cheep is a tiny, single-binary, interactive multi-agent coding shell for your terminal. A lead orchestrator agent breaks a task into pieces, hands each to one or more executor agents (in parallel), verifies the work, and recovers any executor that gets stuck — loops, runs out of context, or errors out. You watch it happen, with a tab per agent.

Orchestrator
Plans, delegates, and verifies. Reads files and runs tests to check the work — but never edits code itself. Best with a strong model.
Executors
Do the actual editing and running, in parallel. Point them at free local models and the work costs nothing.

Provider-agnostic. Any role can be Anthropic (Claude) or any OpenAI-compatible endpoint — local (Ollama, LM Studio, vLLM, llama.cpp) or cloud (OpenAI, Grok/xAI, DeepSeek, Groq, OpenRouter, Mistral, …). Mix an expensive planner with free local workers, or run the whole thing locally.

Why cheep

  • Watch every agent. A tab per agent with live output; see what's delegated to whom and step in any time.
  • See the savings. A running token counter splits usage by orchestrator vs executors and marks local tokens as free.
  • Modes. shift+tab cycles chat → plan (read-only) → auto (full multi-agent delegation).
  • Self-healing. Executors that loop, exhaust their context, or error are detected and recovered automatically.
  • Resumable history. Conversations are saved as transcripts; resume any past session with /history.
  • Zero-config start. First launch discovers local model servers and API keys on your machine and walks you through setup.
  • Tools & MCP. Built-in file/shell tools, plus connect MCP servers to give every agent more capabilities.
  • One static binary. No runtime, no dependencies. brew install, go install, or download.

Best of all worlds

You don't have to pick one model and live with its trade-offs. cheep lets you put the right model on each job: the orchestrator only needs to be smart enough to plan and check the work, and each executor only needs to be good enough for the task in front of it — so you spend premium tokens on judgment and run the bulk of the work cheap, or free and local.

Smart planner, cheap hands
Let a frontier model (Claude, GPT) plan, delegate, and verify, while local models do the editing, searching, and test runs — where they're more than good enough — at no cost.
Right model for the job
Send code to a coding-tuned model, research or drafts to a fast general one, big files to a long-context one. Each executor is just another endpoint — mix and match freely.
All local, zero cost
Point every role at local models for private, offline work that costs nothing — you still get planning, parallelism, and verification on top.
Spend where it counts
A live token counter splits premium vs free usage, so you keep expensive tokens for the hard reasoning and let the long tail of grunt work run cheap.

Quick start

# 1. (optional) run a local model — any OpenAI-compatible server works
#    e.g. Ollama, LM Studio, vLLM, llama.cpp
ollama serve &
ollama pull qwen2.5-coder

# 2. launch cheep
cheep

On first launch cheep opens a setup configurator: it scans your machine for running model servers and API keys, and you pick an orchestrator and at least one executor — or enter cloud credentials manually (Anthropic, OpenAI, Grok, DeepSeek, …). Then you're in the shell. Reopen it any time with /config.

Prefer one-shot, non-interactive runs?

cheep run "add a /health endpoint and a test for it" --workdir .

Configuration

There are no required environment variables. cheep stores its setup in ~/.cheep/config.json and API keys in ~/.cheep/keys.env (0600). You configure agents interactively, not by hand.
